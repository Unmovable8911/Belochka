Belochka (белочка) — инструмент мониторинга серверов в виде единого бинарного файла, предназначенный для небольших групп серверов Linux. Поддерживает постоянные SSH-соединения с 5–20 удалёнными машинами и передаёт метрики CPU, памяти, дисков, сети и процессов в реальном времени на панель управления в браузере через WebSocket. Также предоставляет веб-терминал для прямого SSH-доступа — отдельный SSH-клиент не требуется.

## Возможности

- **Панель мониторинга в реальном времени** — карточки серверов с актуальными метриками CPU, памяти, дисков и сети, окрашенные по уровню нагрузки
- **Детальный просмотр сервера** — индикаторы CPU по каждому ядру, кольцевые диаграммы памяти/swap, детализация по разделам дисков, пропускная способность сетевых интерфейсов, сортируемая таблица процессов
- **Веб-терминал** — полноценная интерактивная SSH-консоль в браузере через xterm.js
- **Иконка в системном трее** — на десктопных машинах (Windows, macOS, Linux с GNOME/KDE/XFCE) отображает иконку в трее с пунктами **Открыть панель** и **Выход**; автоматически переключается в режим CLI на серверах без графической среды
- **Единый бинарный файл** — Go-бэкенд со встроенным React-фронтендом; для развёртывания достаточно одного файла
- **Постоянные SSH-соединения** — автоматическое переподключение с экспоненциальной задержкой и keepalive
- **Шифрование учётных данных** — пароли серверов зашифрованы при хранении с помощью AES-256-GCM
- **Управление cron-задачами** — просмотр, добавление, редактирование, включение/отключение, удаление и немедленный запуск cron-задач прямо на странице сервера
- **Постоянный лог-файл** — весь вывод записывается в лог-файл в каталоге пользовательского кеша (например, `~/.cache/belochka/belochka.log`) с автоматической очисткой по сроку хранения (по умолчанию: 3 дня)
- **Многоязычный интерфейс** — английский, китайский, французский и русский; язык определяется автоматически при первом посещении и может быть изменён в диалоге настроек
- **Встроенные настройки** — настраивайте порт, каталог данных, язык и срок хранения логов прямо из интерфейса через иконку шестерёнки в заголовке панели управления

## Быстрый старт

Скачайте последний бинарный файл со страницы [Releases](https://github.com/Unmovable8911/Belochka/releases) и запустите:

```bash
# Linux (amd64)
chmod +x belochka-linux-amd64
./belochka-linux-amd64

# Windows (64-бит)
belochka-windows-x86-64.exe
```

Откройте `http://localhost:53136` в браузере. Добавляйте серверы через интерфейс.

## Сборка из исходников

Требуется Go 1.25+ и Node.js 18+.

```bash
git clone https://github.com/Unmovable8911/Belochka.git
cd Belochka
make build
./bin/belochka
```

Кросс-компиляция релизных бинарных файлов для всех платформ:

```bash
make release
# Результат:
#   bin/belochka-linux-amd64
#   bin/belochka-linux-arm64
#   bin/belochka-windows-x86-64.exe
#   bin/belochka-windows-x86.exe
```

## Конфигурация

Belochka работает «из коробки» без конфигурации. Все настройки доступны через **диалог настроек** (иконка шестерёнки в заголовке панели управления). При необходимости создайте файл `config.json` в рабочей директории или передайте `--config путь/к/config.json`:

```json
{
  "port": 53136,
  "data_dir": "./data",
  "language": "",
  "log_path": "",
  "log_retention_days": 3
}
```

| Поле | По умолчанию | Описание |
|---|---|---|
| `port` | `53136` | Порт HTTP-сервера |
| `data_dir` | `./data` | Расположение базы данных и ключа шифрования |
| `language` | `""` | Язык интерфейса (`en`, `zh`, `fr`, `ru`); определяется автоматически при первом посещении, если не задан |
| `log_path` | `""` | Путь к лог-файлу; по умолчанию `~/.cache/belochka/belochka.log` |
| `log_retention_days` | `3` | Количество дней хранения записей в лог-файле |

Изменения `port` и `data_dir` требуют перезапуска; `language` и `log_retention_days` применяются немедленно через диалог настроек.

### Флаги

| Флаг | Описание |
|---|---|
| `--config <путь>` | Путь к файлу конфигурации JSON |
| `--no-tray` | Отключить иконку в трее; запустить как CLI-процесс |
| `--version` | Вывести версию и выйти |

### Переменные окружения

| Переменная | Описание |
|---|---|
| `BELOCHKA_ENCRYPTION_KEY` | Ключ AES-256 для шифрования паролей; при первом запуске генерируется автоматически, если не задан |

### Ключ шифрования

При первом запуске без заданного ключа Belochka автоматически генерирует его в `{data_dir}/encryption.key` и выводит предупреждение в логах. Для production-окружения рекомендуется явно задать ключ через переменную окружения.
