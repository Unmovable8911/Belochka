Belochka (белочка) — инструмент мониторинга серверов в виде единого бинарного файла, предназначенный для небольших групп серверов Linux. Поддерживает постоянные SSH-соединения с 5–20 удалёнными машинами и передаёт метрики CPU, памяти, дисков, сети и процессов в реальном времени на панель управления в браузере через WebSocket. Также предоставляет веб-терминал для прямого SSH-доступа — отдельный SSH-клиент не требуется.

## Возможности

- **Панель мониторинга в реальном времени** — карточки серверов с актуальными метриками CPU, памяти, дисков и сети, окрашенные по уровню нагрузки
- **Детальный просмотр сервера** — индикаторы CPU по каждому ядру, кольцевые диаграммы памяти/swap, детализация по разделам дисков, пропускная способность сетевых интерфейсов, сортируемая таблица процессов
- **Веб-терминал** — полноценная интерактивная SSH-консоль в браузере через xterm.js
- **Единый бинарный файл** — Go-бэкенд со встроенным React-фронтендом; для развёртывания достаточно одного файла
- **Постоянные SSH-соединения** — автоматическое переподключение с экспоненциальной задержкой и keepalive
- **Шифрование учётных данных** — пароли серверов зашифрованы при хранении с помощью AES-256-GCM
- **Многоязычный интерфейс** — английский, китайский, французский и русский

## Быстрый старт

Скачайте последний бинарный файл со страницы [Releases](https://github.com/Unmovable8911/Belochka/releases) и запустите:

```bash
# Linux (amd64)
chmod +x belochka-linux-amd64
./belochka-linux-amd64

# Windows (amd64)
belochka-windows-amd64.exe
```

Откройте `http://localhost:53136` в браузере. Добавляйте серверы через интерфейс.

## Сборка из исходников

Требуется Go 1.25+ и Node.js 18+.

```bash
git clone https://github.com/Unmovable8911/Belochka.git
cd Belochka
make build
./bin/belochka
```

Кросс-компиляция релизных бинарных файлов для всех платформ:

```bash
make release
# Результат:
#   bin/belochka-linux-amd64
#   bin/belochka-linux-arm64
#   bin/belochka-windows-amd64.exe
```

## Конфигурация

Belochka работает «из коробки» без конфигурации. При необходимости создайте файл `belochka.yaml` в рабочей директории или передайте `--config путь/к/config.yaml`:

```yaml
port: 53136        # Порт HTTP-сервера (по умолчанию: 53136)
data_dir: ./data   # Расположение базы данных и ключа шифрования (по умолчанию: ./data)
encryption_key: "" # Ключ AES-256; оставьте пустым для автоматической генерации
```

### Переменные окружения

| Переменная | Описание |
|---|---|
| `BELOCHKA_ENCRYPTION_KEY` | Переопределяет значение `encryption_key` из файла конфигурации |

### Ключ шифрования

При первом запуске без настроенного ключа Belochka автоматически генерирует его в `{data_dir}/encryption.key` и выводит предупреждение в логах. Для production-окружения рекомендуется явно задать ключ через файл конфигурации или переменную окружения.
